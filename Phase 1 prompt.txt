## What CommonGround Is

CommonGround is a map-based, open-source community intelligence tool that layers public data sources to surface actionable insights for mutual aid networks, tenant advocates, neighborhood associations, food access organizers, environmental justice advocates, and community organizers. It is built *for* communities, not institutions.

**It is not a surveillance tool.** It is a community nervous system — the capacity to sense what is happening, share awareness laterally, and coordinate response.

---

## Core Non-Negotiables

1. **No individual tracking.** All data surfaces at aggregate level (census tract or zip code minimum). The architecture enforces this technically — raw individual-level records never touch the application database.
2. **No data sales or sharing with law enforcement, immigration enforcement, or any extractive actor.**
3. **No dark patterns, engagement optimization, or advertising. Ever.**
4. **All correlations are labeled as correlations.** Causal claims require primary source evidence. Uncertainty is always named. Every alert carries an explicit claim type: `CORRELATION`, `MECHANISM`, or `DOCUMENTED`.
5. **AGPL-3.0 licensed.** All modifications must remain open source.
6. **Designed for self-hosting and federation.** No proprietary dependencies.

## Alert System

Alerts have three tiers and three claim types:

**Claim types:**
- `DOCUMENTED` — primary source evidence of a specific actor/event; direct language permitted
- `MECHANISM` — correlation + plausible causal pathway supported by external research
- `CORRELATION` — co-occurrence pattern; use "associated with," never "causes"

**Tier 1 (Immediate Action):** T1-H01 Mass Eviction Filing, T1-E01 Severe Weather + High SVI, T1-F01 Community Fridge Critical Need

**Tier 2 (Emerging Pattern):** T2-H01 Compounding Displacement Pressure, T2-F01 Food Access Gap, T2-C01 Civic Action Window, T2-ENV01 Environmental Burden + Vulnerability

**Tier 3 (Situational Awareness):** T3-S01 Assembly Safety Briefing, T3-I01 Surveillance Infrastructure Density, T3-R01 Mutual Aid Gap

Every alert popup must display: claim type, data sources, last updated, and epistemic caution.

## Database Schema (Required)

```sql
layers     — layer metadata, trust rating, claim type, update frequency
features   — geom, layer_id, properties JSONB, aggregation_level, last_updated
alerts     — id, alert_type, claim_type, severity, trigger_conditions JSONB,
             affected_geom, recommendation, caution, created_at, sources TEXT[]
```

**Minimum geometry is census tract.** No parcel-level or address-level storage.

## API Endpoints

```
GET  /layers                          — list layers with metadata
GET  /features?layer=X&bbox=W,S,E,N  — GeoJSON features in bbox
GET  /alerts?bbox=W,S,E,N&severity=X — active alerts for area
GET  /weather?lat=X&lon=Y             — NWS conditions + alerts
POST /resources                       — submit community resource
```

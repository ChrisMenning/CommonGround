## What Changed in Phase 1.5

### 1. Layer Splitting (no re-ingest — data was already in DB)

Four composite data sources were split into focused sub-layers. A `parent_slug TEXT` column was added to the `layers` table. Sub-layers reference their parent via `parent_slug`. When the features API receives a query for a sub-layer, it proxies to the parent's features table rows and returns the full properties JSONB — the client-side CHOROPLETH config selects which property to display.

A `is_composite BOOLEAN DEFAULT false` column was also added. Composite parent layers have `is_composite = true` and are hidden from the sidebar by default. They are accessible via a "Show Composite Layers" toggle.

### 2. OSM Resources — improved categorization

- Removed `pharmacy` from the Overpass query (commercial, not community infrastructure)
- `place_of_worship` retained — many serve as voting locations and community hubs
- Added `category_label` property (human-readable, e.g. "Community Center", "Food Pantry", "Place of Worship") derived from OSM amenity tags
- Feature popup now shows `category_label` prominently in the header instead of raw OSM tag

---

### 3. Layer Grouping Toggle

A toggle above the layer list in the sidebar switches between two grouping modes:

- **By Source** (original Phase 1 behavior): Environmental, Food Access, Housing, Health, Community
- **By Data Type**: Population Indicators, Environmental Burden, Housing Stability, Health Access, Community Infrastructure

Each sub-layer has a `dataType` field in the frontend `LAYER_CONFIG` that determines its Data Type group membership.

---

### 4. Info Drawer (consolidated area-layer info)

Replaced individual floating popups for area/polygon layers with a fixed right-side drawer:

- **Polygon/area layers** (tract, block group, neighborhood aggregation levels): clicking the map opens the drawer with one collapsible card per active layer that has data at that point. First card auto-expands.
- **Point layers** (`fqhc`, `snap-retailers`, `airnow`, `osm-resources`): continue to use the existing MapLibre Popup.
- Each drawer card shows: key metric with choropleth color swatch, claim type badge, source citation, trust rating, and (for the neighborhood-assoc layer) meeting schedule and contact.
- Powered by new API endpoint: `GET /features?point=lng,lat&layers=slug1,slug2`
- Drawer can be dismissed with a close button; re-opens on next map click.

## Database Schema (After Phase 1.5)

```sql
layers (
  id            SERIAL PRIMARY KEY,
  slug          TEXT UNIQUE NOT NULL,
  name          TEXT NOT NULL,
  description   TEXT,
  source_url    TEXT,
  trust_rating  INTEGER CHECK (trust_rating BETWEEN 1 AND 5),
  claim_type    TEXT CHECK (claim_type IN ('DOCUMENTED','MECHANISM','CORRELATION')),
  update_frequency TEXT,
  data_vintage  TEXT,
  parent_slug   TEXT REFERENCES layers(slug),   -- added Phase 1.5
  is_composite  BOOLEAN DEFAULT false,           -- added Phase 1.5
  created_at    TIMESTAMPTZ DEFAULT NOW()
)
```

All other tables (`features`, `alerts`, `resources`) unchanged from Phase 1.

## API Endpoints (After Phase 1.5)

```
GET  /layers                                      — list layers; excludes is_composite=true by default
GET  /layers?include_composite=true               — list all layers including composite parents
GET  /features?layer=X&bbox=W,S,E,N              — GeoJSON features (if layer has parent_slug, proxies to parent)
GET  /features?point=lng,lat&layers=s1,s2,s3     — point-query: features from each layer at a lat/lon point
```

## Key Lessons Carried Forward (Phase 1 + 1.5)

- **MapLibre font names:** Liberty style only: `Noto Sans Regular`, `Noto Sans Italic`, `Noto Sans Bold`. Any other font wipes all dynamically-added layers.
- **Docker compose:** Always `docker compose -f docker/docker-compose.yml ...` from project root.
- **Data vintage:** All new layers must populate `data_vintage`.
- **Ingest pattern:** `const CONFIG = { ... }` block at the top of each seed script.
- **Sub-layer feature queries:** Sub-layers have `parent_slug` set and no own rows in `features`. The API resolves the parent; the frontend CHOROPLETH config selects the display property from the full JSONB.
- **Drawer vs popup:** Drawer = polygon/area layers (multiple can be active). Popup = point layers.
- **Composite layers:** `is_composite = true` hides from sidebar but preserves for signal engine queries.
